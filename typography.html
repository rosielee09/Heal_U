Typography

'Roboto'

Why Roboto?

  • Modern & Professional: Clean, geometric, and approachable.
  • Highly Readable: Optimized for screens, ensuring accessibility.
  • Clear Hierarchy: Uses Regular for body text and Bold for emphasis.
  • Mobile & UI Friendly: Designed for apps, enhancing UX.

Typography Usage

  • Roboto Regular (400): Default body text for descriptions, survey questions, and general content.
  • Roboto Bold (700): Used for section headings, buttons, important messages, and key emphasis.

Text Scales

Style Size/Line Height Example & Usage Why
Heading1 H1 40px / 48px
  • "Hi Rachel, how do you feel?"
  • Onboarding & main page titles
 Large, welcoming, and conversational—sets a friendly, personal tone for first impressions. It commands attention and helps users feel seen, important for an app focused on personal well-being.
Heading2 H2 32px / 40px
  • 'Goal', 'Survey', 'Create an Account'
  • Page Titles
 Strong and clean for clear structure across pages. Slightly smaller than H1 to create hierarchy, but still prominent enough to orient the user within the app.
Section Header H3 24px / 32px
  • "How is your pain today?"
  • Subsection titles
 Guides users through tasks or content sections. It supports scannability and breaks the page into manageable, focused chunks, which is helpful for users experiencing discomfort.
Sub Header H4 20px / 28px
  • "Choose pain points"
  • Survey Instructions
 Instructive but less dominant, so it supports rather than competes with H3. The size is ideal for short, action-oriented prompts.
Body Bold 16px / 24px Button test, emphasized survey answers Makes CTAs and key phrases stand out within paragraphs or forms, improving clarity and accessibility.
Body Regular 16px / 24px
  • "Let's get a new plan tailored to your pain level."
  • Body description
 Standard reading size for comfort and legibility. Works well for general content that users need to read without strain.
Small Text 14px / 20px
  • "14-day plan"
  • Chips
 Small but still readable—ideal for secondary info like labels and tags. Keeps UI clean while providing context.
Tiny Bold 12px / 18px Nav-bar pages text Bold ensures visibility at smaller sizes, critical for navigation. Helps users stay oriented in the app even at a glance.
Tiny 12px / 18px Icons text Pairs well with small icons for labeling. Small footprint keeps the UI uncluttered.
Mini 8px / 12px Numbers for the pain scale component Very small, used only where minimal visual weight is needed—such as numeric indicators. Prioritizes visual space without sacrificing function.
